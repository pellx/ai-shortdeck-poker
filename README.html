🃏 Short Deck LLM Duel Engine / 短牌德州 LLM 对战引擎

让两个大语言模型坐上牌桌，打一场短牌德州扑克。
Pit two LLMs against each other in Short Deck (6+) Hold'em — with live AI commentary.
Python 3.12+ PySide6 GUI MIT License

📖What is this? / 这是什么？

一个桌面 GUI 应用，让两个 LLM（大语言模型）以 AI 玩家身份进行 短牌德州扑克 (Short Deck / 6+ Hold'em) 单挑对战。支持第三个 LLM 担任实时解说员，像电竞赛事一样播报牌局。 A desktop GUI app where two LLMs (large language models) play Short Deck (6+) Texas Hold'em heads-up as AI players. Optionally, a third LLM acts as a live commentator to narrate the match like an esports broadcast.

🚀Quick Start / 快速开始

Prerequisites / 前置条件

  • Python 3.12+Python 3.12+
  • uv — 推荐的 Python 包管理器uv — recommended Python package manager
  • 至少一个 OpenAI 兼容 API（DeepSeek / 豆包 / OpenAI / Gemini via proxy 等）At least one OpenAI-compatible API (DeepSeek / Doubao / OpenAI / Gemini via proxy, etc.)

1. Clone & Install / 克隆与安装

git clone https://github.com/usermbzlj/ai-shortdeck-poker.git
cd ai-shortdeck-poker

# 使用 uv 创建虚拟环境并安装依赖 / Create venv & install deps with uv
uv sync

2. Configure / 配置

cp .env.example .env
# 编辑 .env，填入你的 API 密钥 / Edit .env and fill in your API keys
# AI 选手 A / Player A
A_NAME=DeepSeek
A_BASE_URL=https://api.deepseek.com/v1
A_API_KEY=sk-your-key-here
A_MODEL=deepseek-chat

# AI 选手 B / Player B
B_NAME=Gemini
B_BASE_URL=https://your-api-proxy.com/v1
B_API_KEY=sk-your-key-here
B_MODEL=gemini-2.5-flash

# 解说员（可选）/ Commentator (optional)
COMMENTATOR_BASE_URL=https://api.deepseek.com/v1
COMMENTATOR_API_KEY=sk-your-key-here
COMMENTATOR_MODEL=deepseek-chat

3. Test API / 测试连通性

uv run python test_api.py

4. Run / 启动

uv run python main.py

🎮How to Play / 怎么玩

全自动模式 / Full Auto Mode

  1. 配好 .env 后启动程序Launch the app after configuring .env
  2. 点击 「开始新比赛」Click “Start New Match”
  3. 点击 「▶ 开始自动对局」Click “▶ Start Auto Play”
  4. 坐下来看两个 AI 互相厮杀 🍿Sit back and watch two AIs battle 🍿

手动模式 / Manual Mode

  1. 点击 「开始新比赛」 → 「开始下一手」Click “Start New Match” → “Next Hand”
  2. （可选）手动指定底牌/公共牌(Optional) Manually set hole/community cards
  3. 点击 「随机补齐缺牌」 发牌Click “Fill Missing Cards Randomly” to deal
  4. 用手动操作按钮替 AI 做决策Use manual action buttons to decide for the AI
  5. 每轮行动结束后点 「发下一轮公共牌」 或 「摊牌结算」After each betting round, click “Deal Next Street” or “Showdown”

🃏Short Deck Rules / 短牌规则简介

短牌德州 (6+ Hold'em) 与标准德州的关键差异： Key differences between Short Deck (6+) and standard Texas Hold'em:

  • 36 张牌：移除所有 2-5，最小面值为 636-card deck: removes all 2–5; the lowest rank is 6
  • 牌型排名不同：同花 (Flush) 强于 葫芦 (Full House)Different hand ranking: Flush is stronger than Full House
  • 最小顺子：A-6-7-8-9（A 可作最低牌）Lowest straight: A-6-7-8-9 (Ace can be low)
  • 更多 action：起手牌牌力更接近，弃牌 EV 损失更大More action: starting hands are closer in strength; folding costs more EV

❓FAQ / 常见问题

Q: 支持哪些 LLM？/ Which LLMs are supported?

任何提供 OpenAI 兼容 Chat Completions API 的模型都可以。已测试：DeepSeek、豆包 (Doubao)、Gemini (via proxy)、OpenAI GPT 系列、GLM 等。 Any model that exposes an OpenAI-compatible Chat Completions API works. Tested with DeepSeek, Doubao, Gemini (via proxy), OpenAI GPT series, GLM, etc.

Q: AI 选手 A 和 B 可以用不同的模型吗？/ Can Player A and B use different models?

当然！这正是核心玩法 —— 让不同模型互相对决，看谁扑克水平更高。 Absolutely — that’s the whole point: pit different models against each other and see who plays better poker.

Q: 解说员必须配置吗？/ Is the commentator required?

不是，完全可选。不配置时解说席为空，对局功能不受影响。 No, it’s optional. If not configured, the match still works — there’s just no commentary.

Q: 为什么选短牌而不是标准德州？/ Why Short Deck instead of standard Hold’em?

36 张牌使牌力更接近，AI 需要更多博弈思考而非简单弃牌，对话和决策更有趣。 With 36 cards, hand strengths are closer together. AIs must think more strategically instead of just folding, making decisions and dialogue more interesting.
